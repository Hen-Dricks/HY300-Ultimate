# Appendix F — Glossary

> *"Understanding the terminology is the first step toward understanding the system."*

---

# Purpose

This glossary contains the technical terms used throughout the HY300 Ultimate documentation.

Definitions are intentionally concise while remaining technically accurate. They focus on Android Embedded systems, firmware reverse engineering, Linux internals, and the tools used throughout this project.

This glossary will continue to grow as the project evolves.

---

# A

## A/B Partitions

Android update mechanism that maintains two independent sets of system partitions, allowing seamless updates while preserving the currently running system until the update has been verified.

---

## ADB (Android Debug Bridge)

Official Android debugging interface used to execute shell commands, transfer files, collect logs, and interact with Android devices.

---

## Android Framework

The collection of Java system services and APIs that provide the application runtime environment.

---

## Android Verified Boot (AVB)

Android security mechanism responsible for verifying the integrity of critical partitions during the boot process.

---

## APK

Android application package.

---

## ART (Android Runtime)

The managed runtime used by modern Android versions to execute applications.

---

# B

## Binder

Android's Inter-Process Communication (IPC) mechanism allowing applications and system services to communicate securely.

---

## Block Device

A storage device accessed through fixed-size blocks, such as eMMC, UFS, or SSD partitions.

---

## Boot Image

The image containing the Linux kernel, ramdisk, and boot configuration.

---

## Bootloader

The first software executed after power-on, responsible for initializing hardware and launching the operating system.

---

## Build

A compiled version of software or firmware.

---

## build.prop

A configuration file containing Android system properties read during startup.

---

# C

## Cgroups

Linux kernel feature used to organize and limit groups of processes.

---

## Chroot

A Linux mechanism that changes the apparent root directory of a running process.

---

## CI/CD

Continuous Integration and Continuous Delivery.

Development practices that automate testing and deployment.

---

## CPU

Central Processing Unit.

The primary processor responsible for executing instructions.

---

# D

## Daemon

A background process that continuously provides one or more system services.

---

## Docker

A containerization platform used to provide reproducible development environments.

---

## Dynamic Partitions

Android partition architecture that groups multiple logical partitions inside a single physical partition (`super.img`).

---

# E

## ELF

Executable and Linkable Format.

Standard executable format used by Linux.

---

## e2fsck

Filesystem consistency checker for ext2/ext3/ext4.

---

## ext4

The filesystem used by most Android system partitions.

---

# F

## Fastboot

Android bootloader communication protocol used for maintenance and firmware flashing on supported devices.

---

## Firmware

Software stored in non-volatile memory that controls the operation of a hardware device.

---

## Filesystem

The structure used to organize data stored on a partition.

---

# G

## Git

Distributed version control system.

---

## Git Tag

Named reference used to identify a specific release or milestone.

---

## Ghidra

Reverse engineering framework developed by the NSA for binary analysis.

---

## GPU

Graphics Processing Unit.

Dedicated processor responsible for graphical rendering.

---

# H

## HAL (Hardware Abstraction Layer)

Android software layer separating hardware-specific implementations from the Android Framework.

---

## Hash

Fixed-length cryptographic digest calculated from data.

Used primarily for integrity verification.

---

## HDMI

High-Definition Multimedia Interface.

Digital interface carrying audio and video signals.

---

# I

## Image

A binary representation of an entire partition or storage device.

---

## Init

The first userspace process executed by Android after the Linux kernel starts.

---

## IPC

Inter-Process Communication.

Mechanisms allowing different processes to exchange information.

---

# J

## JADX

Decompiler used to analyze Android APK files.

---

## JNI (Java Native Interface)

Interface allowing Java code to interact with native C/C++ libraries.

---

## Journal

System logs generated by Android or Linux.

---

# K

## Kernel

The Linux operating system kernel used by Android.

---

## Keystone

Image correction mechanism used by projectors to compensate for projection angle distortion.

---

# L

## Launcher

The Android application providing the home screen and application launcher.

---

## Logcat

Android logging utility used to inspect runtime logs.

---

## Logical Partition

Virtual partition stored inside `super.img`.

---

## LP Metadata

Metadata describing Android dynamic partitions.

---

## lpmake

Android tool used to create or rebuild dynamic partition images.

---

## lpdump

Tool used to inspect dynamic partition metadata.

---

## lpunpack

Utility used to extract logical partitions from `super.img`.

---

# M

## Metadata

Information describing the characteristics of data rather than the data itself.

---

## microSD

Removable flash storage medium commonly used for firmware transfer or backup.

---

## Mount

The process of attaching a filesystem to the operating system's directory tree.

---

# N

## Namespace

Linux kernel feature used to isolate resources between processes.

---

# O

## ODM

Partition containing device-specific components supplied by the Original Design Manufacturer.

---

## OEM

Original Equipment Manufacturer.

The company producing or customizing the device.

---

## OTA (Over-The-Air)

Firmware update delivered through a network connection.

---

# P

## Partition

A logical section of storage dedicated to a specific purpose.

---

## Platform

The hardware and software environment supporting an operating system.

---

## Product Partition

Android partition containing product-specific resources and applications.

---

## Process

A running instance of a program.

---

## procfs

Virtual Linux filesystem exposing kernel and process information.

---

# R

## RAM

Random Access Memory.

Temporary working memory used while the system is running.

---

## Recovery

Independent maintenance environment used for updates, repairs, and factory resets.

---

## Reverse Engineering

The process of analyzing a system to understand its design and operation.

---

## Root

The superuser account with unrestricted system privileges.

---

# S

## SELinux

Mandatory Access Control (MAC) framework used to enforce security policies in Android.

---

## SHA-256

Cryptographic hash algorithm used to verify file integrity.

---

## Shared Library

Binary library loaded dynamically by applications or services.

---

## Smali

Human-readable representation of Dalvik bytecode.

---

## SoC (System-on-Chip)

Integrated circuit containing the CPU, GPU, memory controller, and additional hardware components.

---

## Sparse Image

Android image format optimized to reduce storage size by omitting unused blocks.

---

## super.img

Physical partition containing Android logical partitions.

---

## Swap

Storage space used as an extension of physical RAM.

---

## sysfs

Virtual filesystem exposing Linux kernel objects and hardware information.

---

## System Partition

Primary Android partition containing the operating system.

---

# T

## Tombstone

Crash report generated after a native process failure.

---

## TXCZ

Name associated with several proprietary components identified within the HY300 firmware.

---

# U

## UID

User Identifier assigned by the Linux kernel.

---

## USB Debugging

Android feature enabling ADB communication over USB.

---

# V

## vbmeta

Partition storing Android Verified Boot metadata.

---

## Vendor Partition

Partition containing vendor-specific hardware components and drivers.

---

## Volume

Logical storage entity composed of one or more partitions.

---

# W

## Working Copy

A duplicate of the original firmware used for experimentation.

---

# Z

## ZRAM

Compressed block device residing in RAM and commonly used as compressed swap memory.

---
